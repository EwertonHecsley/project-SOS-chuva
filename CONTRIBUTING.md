# Guia de Contribuição 🤝

Obrigado por se interessar em contribuir para o **SOS Chuva**! Este projeto é focado em ajudar pessoas em situações críticas, e sua ajuda é fundamental.

## Como posso contribuir?

### Reportando Bugs
- Verifique se o bug já não foi reportado nas **Issues**.
- Se não, abra uma nova issue descrevendo o problema, passos para reproduzir e seu ambiente (browser, OS).

### Sugerindo Funcionalidades
- Novas ideias são sempre bem-vindas! Abra uma issue com a tag `enhancement`.

### Pull Requests
1. Escolha uma issue para trabalhar ou crie uma própria.
2. Faça um fork do repositório.
3. Crie uma branch para sua alteração (`git checkout -b feature/minha-feature`).
4. Faça o commit de suas alterações (`git commit -m 'feat: adiciona nova funcionalidade'`).
5. Faça o push para a branch (`git push origin feature/minha-feature`).
6. Abra um Pull Request.

## Padrões de Código

- **Linguagem:** O código e comentários devem ser em Inglês (padrão da indústria), mas a interface e documentação principal em Português.
- **Componentes:** Use componentes funcionais com Hooks.
- **Estilização:** Use Tailwind CSS e siga o design system existente.
- **Commits:** Tente seguir o padrão de [Conventional Commits](https://www.conventionalcommits.org/).

## Configuração do Ambiente

1. Certifique-se de ter o `pnpm` instalado.
2. Siga as instruções no `README.md` para rodar o projeto localmente.
